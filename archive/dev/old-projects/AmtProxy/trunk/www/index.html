AMT RTSP Proxy Server

The AMT RTSP Proxy Server can be started as an independent process via Java Web Start. Once the server has been started, RTSP clients can use it to access a multicast streaming media presentation. The proxy server listens for RTSP requests on port 8054. The HTTP URL for an SDP description for a stream must be passed as the query string value in the RTSP URL. For example:

rtsp://127.0.0.1:8054/?<sdp-url>

On Windows, Java Web Start will start a windowless process with the name "javaws.exe". At present, it can only be stopped by killing the process directly. On Mac OS X, the server will run as a minimal application with a menu that can be used to stop it.

The proxy server does not accept SDP descriptions for unicast presentations. The connection records in the SDP must specify multicast addresses and any SDP description that contains an SSM connection address must include a source-filter attribute record that applies to that address. The proxy server can only fetch SDP descriptions using from an http: URL - it cannot yet act as an RTSP client to fetch SDP descriptions from another RTSP server.
